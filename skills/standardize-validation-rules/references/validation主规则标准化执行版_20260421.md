# Validation 主规则标准化执行版

适用文件：

- `~/Downloads/20260415134001-t_1.xlsx`

这份文档只负责 `validation` 主规则标准化，不包含 mitigation 字典补数细节。

---

## 一、适用范围

适用于以下内容：

- `Actions`
- `Sequences`
- `Playbook`

不适用于以下内容：

- `mit_1.xlsx` 的 remediation 补数
- mitigation 字典结构定义
- CVE 详情抓取与 reference 补链流程

---

## 二、总目标

把 validation 主表统一成“验证动作 / 验证场景 / 剧本”口径，并按规则类型统一标题、描述、notes 的表达方式。

---

## 三、通用标准化总则

### 1. 标点与格式

- 所有中文顿号统一改为中文逗号 `，`
- 标题中的 `空格 - 空格` 按场景调整为中文逗号结构
- `variant` / `变种 1` / `Variant-1` / `Variant #1` 统一成 `变种 #1`
- 删除 `- [*] -` 这类无效结构
- 清理 `_x000D_`
- 如果句子缺少停顿或标点，补足使中文通顺

### 2. 固定词替换

- `CJIAJIA` / `C+JIAJIA` -> `C++`
- `威胁行为体` -> `威胁组织`
- `威胁集群` -> `威胁组织`
- `网络间谍组织` -> `威胁组织`
- `网络间谍活动集群` -> `威胁组织`
- `行为体` -> `组织`
- `该操作` -> `该验证动作`
- `此操作` -> `此验证动作`

### 3. 链接与参考

如果正文后需要补参考链接，统一格式为：

```text

请参考：
https://...
https://...
```

规则：

- 链接区前空一行
- 如果 reference 中是可点击文本但真实 URL 在 address 中，取真实 URL
- 最多保留前 5 个链接

### 4. 中国相关表述修正

- 包含“中国”的威胁组织描述，要删除中国相关限定并校正句子

---

## 四、Actions 标准化

### 当前字段

- `uuid`
- `vid`
- `cn_name`
- `en_name`
- `cn_desc`
- `en_desc`
- `cn_notes`
- `en_notes`
- `created`

### 总体目标

把 `Actions` 统一成“验证动作”口径。

### 1. Web 类规则

#### 标题规则

- `此检测规则` 改成 `此验证动作`
- `Web安全验证 - Web应用程序漏洞` 时，去掉前面的 `Web安全验证`
- 若原规则被 detection 过度化，改回 `Web应用程序漏洞`
- 若判断是硬件设备或操作系统类目标，标题前面的 `Web` 去掉
- 标题中的 `空格 - 空格` 改为中文逗号结构
- `CVE` 前移并加中文逗号

#### 描述规则

描述统一改写为：

- 以 `此验证动作还原了针对...` 开头
- 句式统一为 `此验证动作还原了针对 XXX，存在的 YYY 漏洞的利用尝试`
- 中文需校正到通顺，不机械硬套模板

补充规则：

- 前面的软件描述剪切到后面
- 软件描述精简后置
- 参考链接区按统一模板后置

#### 类型判断

- 没有关联软件、厂商的 `Web应用程序漏洞` 规则，改归为 `Web安全验证`

### 2. 恶意文件传输

标题统一为：

`恶意文件传输 - APT组织，恶意文件名称或类型，下载，变种 #数字`

规则：

- APT 名称保留
- 恶意文件名或类型尽量从描述中抽取
- `下载` 动作补齐
- 若恶意文件名判断不稳，需标黄复核

### 3. 主机命令行

- 从 tag 或 notes 读取操作系统
- 若没有变种编号，则把 OS 类型放在规则名称后方
- 若有变种或下载动作，则 OS 放在对应字段前面

### 4. 邮件

标题统一为：

`钓鱼邮件 - 恶意链接/恶意附件，APT组织，文件名或类型（如果有），变种 #数字`

补充规则：

- `subject` 根据热点生成

### 5. 受保护的沙盘

标题统一为：

`受保护的沙盘 - 组织，文件，执行，变种 #数字`

固定词：

- `受保护的剧场` / `受保护剧场` -> `受保护的沙盘`

### 6. 命令与控制

固定词：

- `电报通信` -> `Telegram 通信`
- `任务` -> `任务流量`
- `签到` -> `签入`

### 7. notes 列

统一替换：

- `源 actor` -> `源验证机器人`
- `目标 actor` -> `目标验证机器人`

## 五、Sequences 标准化

### 当前字段

- `uuid`
- `vid`
- `cn_name`
- `en_name`
- `cn_desc`
- `en_desc`
- `created`

### 总体目标

把 `Sequences` 统一成“验证场景”口径。

### 标题规则

- 标题统一为 `恶意活动场景 - APT组织 攻击活动`
- 删除 detection 口径和杂糅的动作式命名

### 描述规则

- 统一以 `此验证场景包括了...` 开头
- 描述多个动作或步骤时，保留顺序关系，但句式统一

---

## 六、Playbook 标准化

### 标题规则

- 标题统一按 `威胁组织` / `APT组织` 口径

### 描述规则

- 统一以 `此剧本模拟了...` 开头
- 保留攻击目标、攻击链条和关键行为，不保留 detection 叙述方式

---

## 七、与 mitigation 的关系

- `t_1.xlsx` 需要 remediation 时，只复用已经确认的 mitigation 输出
- 不在主规则标准化阶段自由生成 remediation 文本
- 若 `Actions` 与 mitigation 结果冲突，以 mitigation 已确认结果为准
