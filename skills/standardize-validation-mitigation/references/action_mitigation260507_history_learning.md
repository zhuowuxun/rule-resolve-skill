# action_mitigation260507 历史标准化学习

来源文件：`/Users/carmenz/Downloads/action_mitigation260507.xlsx`

该文件是已标准化的 validation mitigation 历史语料，结构为：

- `uuid`
- `name`
- `mitigations_cn`
- `mitigations_en`

这不是 `mit_1.xlsx` 原始结构，也不是 `Actions / Sequences / Playbook` 命名标准化规则。只用于学习和校验 mitigation/remediation 口径。

## 使用原则

- 只能用于 mitigation 分支，不能把这里的 `name` 前缀或标题写法同步到 validation 主规则标准化。
- 历史语料可以作为已确认 remediation 口径参考。
- 若当前 mitigation 行的 `cn_notes` / `en_notes` 为空或为 `#N/A`，且历史语料中存在相同 `uuid` 或完全相同 `name`，可以使用历史语料中的 remediation 作为高置信回填来源。
- 使用历史语料回填时，只取 remediation 主体段落。不要复制历史语料中已有的 CVE 描述或 `请参考：` 链接块；CVE 描述和 reference 必须按当前 CVE 重建。
- 如果当前行已有 remediation 文本，只能按规则清理和标准化，不要用历史语料覆盖。
- 无精确 `uuid` / `name` 命中时，不根据相似标题直接复制历史行，只能把历史语料作为模板口径参考。
- 历史回填属于自动补数，需要和字典回填一样标黄，并写入 `自动回填记录`。

## 高置信 mitigation 口径

### 恶意文件传输

主流口径是网络安全产品 + 传输文件 Hash：

```text
如果网络安全产品对此攻击漏检，塞讯验证短期建议：基于传输的恶意文件的Hash在网络安全产品中添加黑名单。中长期建议：评估沙箱产品与网络安全类产品联动；加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御，在预算可行的情况下全面部署NTA（全流量）产品加强威胁检测。
```

若标题看起来是“下载恶意文件”但规则类型是网络传输，不要套用主机执行/终端权限模板。

### 命令与控制

优先使用网络检测口径。常见两类：

```text
如果网络安全产品对此攻击漏检，塞讯验证短期建议：在互联网边界以及各个安全区域之间部署NIPS加强检测与防御。中长期建议：评估威胁情报以加持相关安全产品提升对此类通信的C&C服务器的识别率。
```

```text
如果网络安全产品对此攻击漏检，塞讯验证短期建议：加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御。中长期建议：增加DNS安全解决方案。
```

若规则是 DNS 恶意域名查询，可补充 DNS 安全解决方案；若是 C&C 流量、信标、隧道、反连，优先 C&C/NIPS 口径。

### 钓鱼邮件

钓鱼邮件使用邮件安全网关口径，不使用网络/主机安全产品泛化口径。

恶意附件/文件：

```text
如果邮件安全网关对此钓鱼邮件漏检，塞讯验证短期建议：基于此恶意文件的Hash添加邮件安全网关的本地黑名单策略。中长期建议：增加沙箱与邮件安全网关联动；加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御；提高全员钓鱼演练频率，加强员工安全意识培训。
```

恶意链接：

```text
如果邮件安全网关对此钓鱼邮件漏检，塞讯验证短期建议：基于此恶意链接URL添加邮件安全网关的本地黑名单策略。中长期建议：严格执行密码策略；增加各种应用系统登录与MFA联动；增加沙箱与邮件安全网关联动；加强互联网访问的管控与下载文件检测；提高全员钓鱼演练频率，加强员工安全意识培训。
```

### 受保护的沙盘

`此攻击手法是具有破坏性的` 只能保留在 `受保护的沙盘` 类规则中。

历史语料和旧字典中有少量 `危险性的` / `dangerous` 旧口径。沙盘类统一纠正为 `破坏性的` / `destructive`，非沙盘行残留的破坏性或危险性句式都需要清理，不是可学习规则。

### Web / 应用程序漏洞

普通 Web / 应用程序漏洞默认使用：

```text
如果网络或主机安全产品对此攻击漏检，塞讯验证短期建议：针对此漏洞打补丁修复或升级软件；通过优化WAF产品的检测规则实现防御或评估RSAP产品在贵司的适用性。中长期建议：增加特权帐号管理提升身份安全；增加多因素认证；加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御；做好应用程序隔离。
```

若是硬件设备、网络设备、安全设备、VPN、网关、路由器、防火墙、负载均衡等产品，应改为网络安全产品口径，并去掉：

- `或主机安全产品`
- `通过优化WAF产品的检测规则实现防御或评估RSAP产品在贵司的适用性`
- `做好应用程序隔离。`

历史语料中确认过的网络/安全设备示例包括：

- `深信服下一代防火墙`
- `安恒明御WEB应用防火墙`
- `F5 BIG-IP`
- `Cisco ASA 和 Firepower`
- `Cisco vManage`
- `Cisco Smart Licensing Utility`
- `Cisco IOS 和 IOS XE 集群管理协议（CMP）`
- `Pulse Secure SSL VPN`
- `Ivanti Connect Secure VPN`
- `Ivanti Endpoint Manager Mobile`
- `Fortinet FortiMail`
- `D-Link`
- `D-Link Central WiFiManager 软件控制器`
- `Buffalo 路由器`
- `飞鱼星路由器`
- `腾达 FH1201 路由器`
- `网神SecGate 3600防火墙`
- `WIFISKY-7层流控路由器`
- `瑞斯康达 多业务智能网关`

注意：不要把 `WAF绕过` 前缀当作硬件设备规则。`WAF绕过` 规则仍保留 WAF/RSAP 相关建议。

### 云上系统 / 云软件

历史语料中，`孚盟云`、`用友 U8 Cloud`、`金蝶云星空`、`极简云验证`、`百易云资产管理运营系统` 等云上业务系统或云软件，主流 remediation 是网络安全产品口径，不使用 `网络或主机安全产品`。

处理规则：

- `网络或主机安全产品` 改为 `网络安全产品`
- 删除 `通过优化WAF产品的检测规则实现防御或评估RSAP产品在贵司的适用性`
- 删除 `做好应用程序隔离。`
- 无 CVE 的厂商云产品优先使用厂商告知口径，例如：

```text
如果网络安全产品对此攻击漏检，塞讯验证短期建议：联络孚盟云告知漏洞。中长期建议：增加特权帐号管理提升身份安全；增加多因素认证；加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御。
```

不要把所有英文 `Cloud` 字样都机械套用该规则。`Spring Cloud`、`Cloudlog`、`VMware Cloud Foundation`、`ownCloud`、`Nextcloud` 等仍需结合历史语料和产品形态判断。

### AI 应用程序漏洞

历史语料里曾出现过更宽的防护对象：

```text
如果网络或主机或AI应用安全产品对此攻击漏检
```

当前不学习这个旧口径。AI 应用程序漏洞默认仍使用通用 `网络或主机安全产品` / `network or server security products`，不要主动扩写为 `AI应用安全产品`。

### 数据泄露 / DLP

数据泄露类使用 DLP 口径：

```text
如果网络DLP产品对此数据泄露动作漏检，塞讯验证短期建议：检查网络DLP的策略是否开启对此协议的检测。中长期建议：排查网络DLP的覆盖度。
```

HTTPS/SSL 场景可加入 SSL 解密设备相关建议。

### OT / 工控安全

OT 类漏洞使用 OT 安全设备口径：

```text
如果网络安全产品对此攻击漏检，塞讯验证短期建议：在工控环境中旁路部署OT安全设备进行资产识别；做好环境中各安全区域的访问控制；关注设备厂商安全公告并及时安装补丁程序。中长期建议：加强互联网访问的管控与下载文件检测，并加强内网网络流量的威胁文件传输检测；在内部各个安全区域之间部署NIPS加强检测与防御，在预算可行的情况下全面部署NTA（全流量）产品加强威胁检测。
```

### 容器安全

容器类规则使用容器或主机安全产品口径：

```text
如果容器或主机安全产品对此攻击漏检
```

常见建议包括容器系统加固、升级镜像修补漏洞、HIDS 或容器安全产品规则更新、容器实时文件落盘扫描。

## CVE 描述和 reference

历史语料中存在旧格式的 CVE 描述/reference 块，部分行的 `请参考：` 放置顺序不稳定。

标准化时不要复制历史尾部块，必须：

- 保留 remediation 主体段落在最前
- 删除旧 appended description/reference block
- 用当前 CVE 来源重建描述和 reference
- 中文链接块固定为：

```text

请参考：
https://...
https://...
```

- 英文链接块固定为：

```text

Please refer to:
https://...
https://...
```
